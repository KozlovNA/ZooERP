## Технологии и Принципы
*   **Язык:** C++
*   **Сборка:** CMake
*   **Тесты:** GoogleTest (LCOV покрытие ~80%)
*   **DI-контейнер:** Boost.DI 

### Архитектурные решения (SOLID)
1.  **SRP:**
    *   `Zoo` отвечает только за хранение данных.
    *   `MenuSystem` отвечает только за взаимодействие с пользователем.
    *   `VetClinic` отвечает только за проверку здоровья.
2.  **OCP:**
    *   Новые виды животных добавляются созданием новых классов, наследуемых от `Herbo` или `Predator`, без изменения кода `Zoo`.
    *   Меню реализовано через паттерн **State**, что позволяет легко добавлять новые экраны.
3.  **LSP:**
    *   Везде, где используется `Animal*` или `IInventory*`, можно подставить любого наследника без нарушения логики программы.
4.  **ISP:**
    *   Разделены интерфейсы `IAlive` и `IInventory`. Животные реализуют оба, вещи — только `IInventory`.
5.  **DIP:**
    *   Класс `Zoo` не зависит от конкретной `VetClinic`, а зависит от абстракции `IVetClinic`. Внедрение зависимости происходит через **Boost.DI**.

## Инструкция по запуску

### Предварительные требования
*   CMake (3.14+)
*   Компилятор C++ (GCC/Clang/MSVC) с поддержкой C++17
*   Интернет (для скачивания зависимостей Boost и GTest)

### Сборка и Запуск (Linux/macOS)
```bash
mkdir build
cd build
cmake ..
cmake --build .

# Запуск приложения
./ZooApp

# Запуск тестов
./ZooTests
```
